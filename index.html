Cloud Transparency Initiative

About Us

The Cloud Transparency Initiative is a product of the University of Washington's iSchool Capstone Program, created by students Jordan Atwood and Doug Wynkoop.

The Problem

Company networks stand to benefit greatly through cloud service adoption, but improperly-chosen or ill-established cloud services can be insecure or unstable.

Our Solution

For lack of a centralized reference or resource, we created this website to highlight key concerns and address anticipated problems when considering cloud solutions.

Threats & Solutions

In securing any network, there are a number of attack surfaces which must be considered.

We will cover issues pertaining to these three main surfaces:

Local Network
Threats which concern the physical network owned by the company itself, particularly relevant if no cloud solution is utilized.
Cloud Network
Issues regarding communication between the company's network and the cloud provider's network, as well as concerns regarding the security and integrity of the provider's network.
Legal & Compliance
There are a wide variety of compliance and regulatory rules that can affect the administration of a network and its security.

A final note before diving into each surface: while verifying that these issues are resolved, being able to trust your network provider (cloud or otherwise) is vitally important. They will be handling all of your data, and should thus be responsible for its safety and protection. That, above all else, should be prioritized.

Network

Unsecured Ports

Network communication occurs by sending information through a specific pathway to a computer. These pathways are called ports, and should be taken into consideration when securing a network from outside traffic.

Computers have thousands of ports, most of which are assigned to traffic for a particular application. Other ports, on the other hand, are used for specific network protocols, and only certain types of data are expected to travel through them.

Securing Ports

Because most ordinary network traffic will be carried out by a core set of applications (web browsers, file management clients, or proprietary software), it is typically wise to disallow traffic which attempts to communicate over ports which are not used by those applications — that is, to whitelist safe traffic, blocking all other traffic. The port used for a given application can be found using a network traffic analyzer or by referencing a list of port numbers.

Enabled Services

In the same vein as unsecured ports, computer systems will typically come installed with a suite of applications of various uses — productivity software, system management, and more. Because these applications are already installed on the system, a malicious outsider would only need to gain access to one which could significantly damage or compromise a system rather than attempt to install malware or other invasive programs.

Fortunately, mitigating the risk of enabled services is as simple as uninstalling programs and services which are not used, and limiting the use of those which are installed by use of user management, or some other permissions system.

Man-in-the-middle Attacks

Man-in-the-middle attacks are attempts to listen in on, or interrupt network traffic in transit, and are typically carried out over wireless mediums, as traffic is sent over the air, rather than directly from peer to peer. Such attacks aim to glean sensitive information as it is sent over the network, such as passwords, personal information, and financial information.

Because these types of attacks rely on receiving data which is human-readable, the primary method to combat them is to encrypt all network traffic in transit. This will mean that websites will be served over https (HTTP network traffic on top of an SSL or TLS protocol) or some other secure protocol. Any implementation of this type of encryption should be applied to all pages of a website in order to prevent any would-be attackers from stealing a logged-in user's session ID and using it to log into that website as the victim.

Insider Threat

While various company members may have reason to need access to a wide variety of network resources, it is in the best interest of security not to allow all employees universal permissions. Instead, permissions should be given to groups, and users assigned to the group which fits their needs. That way, a malicious insider would have significantly limited access compared to a universal "all-access-granted" permissions system.

Cloud

Data Storage

When storing information in the cloud, there are a number of issues which present themselves in how the data is stored. Regarding storage, data in the cloud should always be encrypted so that it cannot be deciphered if stolen or hijacked.

Beyond physical protections, cloud providers should additionally provide assurances which protect data from themselves. For instance, data uploaded to the cloud should not be used by the provider for data analysis, and should be safe from insider access. Furthermore, data which is deleted or data which was stored on equipment removed from the cloud environment should not remain in memory or on disk — it should be permanently and irreversibly purged from the cloud system. (except in special cases, see eDiscovery)

Data Recovery and Data Loss

In addition to ensuring cloud data is encrypted (thus protected from data theft), data stored on the cloud should also be redundant and accessible or should otherwise be able to be recovered in the case of accident or equipment failure.

Network Visibility

Cloud networks are comprised of a variable number of physical or virtual machines all cooperating, in a sense, to store and compute data. Because of this, they are significantly more complex than a traditional physical network and may have exposed vulnerabilities due to misconfigured or improperly-installed machines. As such, cloud providers should be proactive regarding these issues and forthcoming about measures they take to combat them.

Governance

Data storage is a practice which has a number of laws written regarding how and where it can be stored, who can access it, etc. In addition, these laws are not identical between states and countries. As such, cloud providers should not withhold data center locations, and should make clear any implications which come with data being stored at a particular location.

Compliance

PCI DSS

PCI DSS is the industry standard for storage and use of credit card-related information. Violation can result in fines and/or get a company suspended from accepting credit cards for payment. When dealing with any cloud provider, the requirements to meet these standards are less clear, and should be explicitly discussed to avoid confusion.

While each company may have its own standards, there are 12 major stipulations they require:

  • Build and Maintain a Secure Network
  • Change Vendor Defaults
  • Protect Data
  • Encrypt Transmission of Cardholder Data
  • Use and Regularly Update Anti-Virus Software
  • Develop and Maintain Secure Systems and Applications
  • Restrict Access to Cardholder Data
  • Assign a Unique ID to Each Person with Computer Access
  • Restrict Physical Access to Cardholder Data
  • Track and Monitor All Access to Network Resources and Cardholder Data
  • Regularly Test Security Systems and Processes
  • Maintain a Policy that Addresses Information Security for all Personnel

eDiscovery

eDiscovery is the legal requirement that a company provide electronic information during an inspection or civil suit, and is in place to provide a method that an entity can use to prove that they have been acting in good faith and following data storage and protection laws.

eDiscovery typically will require three things of organizations: an audit trail, information preservation — that stored data be preserved for a certain amount of time before it may be deleted, and organization and metadata — information regarding the category, location, network backups, and more of specific data.

Sarbanes-Oxley

Sarbanes-Oxley laws require that companies be responsible for financial data and records, and must provide documentation that they are sufficiently protecting them, such as:

Security Policies
Companies must provide documentation on all policies designed to keep their network safe.
Meeting Industry Benchmarks
Companies must prove they are meeting industry benchmarks for all of the following:
  • Workstation and server configuration
  • Physical security
  • Network infrastructure administration
  • System access controls
  • Data classification and management
  • Application development and management
IAAA
Companies must prove that they are providing adequate Identification, Authentication, Authorization, and Account management.
Network Security
Companies must prove they are taking adequate care of their networks, utilizing a good network perimeter, encryption, antivirus, and strong wireless security.
Monitoring
Networks must be able to be monitored.
Segregation of Duties
Network employees must have segregated duties, and with them, segregated permissions.
Physical Security
Cloud providers' data centers must be highly physically secure as to prevent physical data breach.
